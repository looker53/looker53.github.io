session和token的区别

· 2021-01-16 · # 计算机基础

session

session 消耗太多存储资源

当碰到大量的用户，需要用到服务器集群和跨域。 数据同步和session共享。

token

就是要解决不需要在服务端存储，也可以识别用户。

而解决问题的关键，就是签名。

token 不能强制失效

session 是可以强制失效的，直接删除服务端的用户数据。

token 是统一印发的，每张除了用户信息和过期时间不一样，都是相同的。

如果因为一些特殊原因想让令牌强制失效，也需要在服务端保存。 这样每次在服务端，还是要存储令牌数据，只是比 session 会少一些。

token 可以存在 cookie 中吗？

可以

token 是怎么生成的
